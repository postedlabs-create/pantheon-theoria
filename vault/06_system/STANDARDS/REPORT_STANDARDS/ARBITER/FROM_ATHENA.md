# ARBITER — Acceptance Criteria for Athena Reports

*What Arbiter expects to receive from Athena each cycle. Athena reads this to know exactly what Arbiter requires.*

---

## Delivery Requirements

Athena delivers **three documents** to Arbiter each cycle. All three must arrive together. Partial delivery is not accepted.

---

## Document 1: Unified Executive Summary

**Format file:** ATHENA/TO_ARBITER_SUMMARY.md

**Purpose:** Top-level synthesis for Arbiter's situational awareness.

**Required sections:**
- Division Status (one sentence per active agent)
- Top Signals (3-5 intelligence items that require Arbiter attention)
- Recommended Actions (concrete, prioritized -- not suggestions, recommendations)
- Open Issues / Blockers

**Format standards:**
- Maximum 1 page
- Bullet points only -- no prose paragraphs
- Every item must be actionable or informational -- no filler
- Date/cycle clearly labeled at top

---

## Document 2: Daily Strategic Report

**Format file:** ATHENA/TO_ARBITER_STRATEGIC.md

**Purpose:** Deeper operational read on intelligence gathered and strategic posture.

**Required sections:**
- Intelligence Summary (what Argus and other ops agents surfaced)
- Competitive Landscape (relevant POSTED market signals)
- Outreach Activity Summary (Calliope performance)
- Strategic Recommendations (specific, directional)
- Metrics (any available -- engagement, outreach volume, response rate, etc.)

**Format standards:**
- Structured with clear headers
- Quantitative where possible -- no vague qualitative assessments without backing data
- Date/cycle labeled at top

---

## Document 3: Field Agent Assessment

**Format file:** ATHENA/TO_ARBITER_ASSESSMENT.md

**Purpose:** Agent-level performance review for Arbiter's tuning pipeline.

**Required sections (per agent):**
- Agent Name & Role
- Output Quality (1-5 scale + one-line justification)
- Alignment with Mission (on-track / drifting / misaligned)
- Notable Issues (specific, not generalized)
- Athena's Tuning Recommendation (if any)

**Format standards:**
- One section per active agent
- Specific -- no generic praise or vague criticism
- If no issues, say so plainly

---

## Non-Negotiables

- All three documents delivered together each cycle
- No hedging language ("it seems," "possibly," "might")
- Data cited where claims are made
- Documents returned for revision if standards not met
