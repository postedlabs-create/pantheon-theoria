# REPORT_FORMATS.md — Document Structures for Arbiter

All three documents are delivered to: `Pantheon/Theoria/Commander/`  
File naming convention: `YYYY-MM-DD_[DocType].md`

---

## Document 1 — Unified Executive Summary

**Purpose:** Single consolidated picture of where the division stands. Arbiter reads this first.  
**Time to consume:** 2 minutes maximum.  
**Lead with:** What matters most, not what happened first.

```
# Unified Executive Summary — [Date]
**Cycle:** [Cycle number or date range]
**Prepared by:** Athena | Chief Strategist

## Division Status: [GREEN / YELLOW / RED]

[3–5 sentences. The single most important thing happening across the division. What it means. What Arbiter needs to know right now.]

## Key Numbers This Cycle
| KPI | Value | Trend |
|-----|-------|-------|
| Leads identified (Argus) | # | ↑↓→ |
| Outreach sent (Calliope) | # | ↑↓→ |
| Response rate | % | ↑↓→ |
| Conversion rate | % | ↑↓→ |
| Competitor mentions tracked | # | ↑↓→ |

## Top Priority for Arbiter
[One sentence. The single decision or action that matters most this cycle.]

## Agents at a Glance
| Agent | Status | Flag |
|-------|--------|------|
| Argus | Exceeding / Meeting / Needs Attention / Underperforming | [issue if any] |
| Calliope | Exceeding / Meeting / Needs Attention / Underperforming | [issue if any] |
```

## Document 2 — Daily Strategic Report

**Purpose:** Full operational and strategic picture across 8 sections.

```
# Daily Strategic Report — [Date]
**Cycle:** [Cycle number or date range]
**Prepared by:** Athena | Chief Strategist

---

## Section 1: Division Activity
[What is happening across the division right now. Factual. Operational. No analysis yet.]

## Section 2: What Matters Most
[Of everything in Section 1, what actually changes the trajectory of the mission. Why it matters.]

## Section 3: Decision Points
[What requires Arbiter's awareness now — not decisions yet, just context for Section 6.]

## Section 4: Risks and Threats
[What could go wrong. Probability where estimable. Impact if it does. Mitigation options.]

## Section 5: Emerging Patterns and Insights
[Signals that don't yet require a decision but indicate where things are heading. Patterns across agents.]

## Section 6: Decisions Required ⚠️
[THE MOST IMPORTANT SECTION. Be specific. Format:]

**Decision [#]: [Title]**
- Situation: [What is happening]
- Options: [A / B / C with brief tradeoffs]
- Recommendation: [Athena's recommended path]
- Deadline: [When this must be decided]

## Section 7: Actions in Motion
[What is currently being executed across the division. Who owns it. Expected completion.]

## Section 8: Watch List — Next Cycle
[Signals to monitor. Thresholds that would trigger escalation. What Athena will be looking for.]
```

## Document 3 — Field Agent Assessment

**Purpose:** Structured evaluation of every active agent. Arbiter uses this to identify who needs optimization.

**Effectiveness Ratings:** Exceeding / Meeting / Needs Attention / Underperforming

```
# Field Agent Assessment — [Date]
**Prepared by:** Athena | Chief Strategist

---

## [Agent Name] | [Role]

**Effectiveness Rating:** [Exceeding / Meeting / Needs Attention / Underperforming]

**Strengths:**
- [Specific, evidence-backed]

**Weaknesses:**
- [Specific, evidence-backed]

**Strategic Alignment:**
[Is this agent's current activity aligned with the division's priority objectives? Where is the gap if any?]

**Recommendation:**
[Concrete. Actionable. Directed at Hermes or Arbiter as appropriate.]

---
[Repeat for each active agent]

## Assessment Summary Table
| Agent | Rating | Primary Strength | Primary Gap | Action Required |
|-------|--------|-----------------|-------------|-----------------|
| Argus | | | | |
| Calliope | | | | |
```
