# OPERATIONS → COMMAND: Communication Standard

*What Operations must report vs resolve independently. Intelligence flow upward.*

---

## Report vs Resolve

### Operations resolves independently:
- Routine task execution within defined scope
- Minor execution adjustments within existing mission parameters
- Expected outputs delivered on schedule

### Operations must report immediately:
- Unexpected findings with strategic implications
- Blockers that prevent mission completion
- External signals that change the operating picture
- Anything outside defined mission scope that requires commander decision

---

## Urgency Levels

| Level | Trigger | Action |
|-------|---------|--------|
| CRITICAL | Threat, breach, time-sensitive opportunity | Report immediately — interrupt cycle |
| HIGH | Significant blocker or strategic signal | Report within current cycle |
| STANDARD | Scheduled output / routine intelligence | Report at end of cycle |
| LOW | Background observations | Include in next scheduled report |

---

## Format Headers (All Operations Reports)

```
AGENT: [Name]
DATE: [YYYY-MM-DD]
CYCLE: [Cycle number or date]
URGENCY: [CRITICAL / HIGH / STANDARD / LOW]
ROUTED TO: [Archon MCP]
SUBJECT: [One-line summary]
---
[Body]
```

---

## Intelligence Flow Path

Operations → Archon MCP → Command (Athena for synthesis, Hermes for agent issues)

Operations does NOT route directly to Arbiter. Everything passes through Archon MCP.

---

## Response Time Expectations

- CRITICAL: Immediate
- HIGH: Within 2 hours
- STANDARD: Per cycle schedule
- LOW: Next scheduled report
